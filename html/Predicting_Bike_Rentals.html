Predicting the number of bikes rented per hour¶

In this project, we use a few simple machine-learning models (linear regression, decision tree, and random forest) to predict the hourly number of bikes rented in Washington, D.C.. Like many other American cities, Washington, D.C. has a communal bike-renting service. Bikes can be rented for either one hour or one day.

One important problem for cities setting or and maintaning similar services is to estimate how many and where bicycles are needed. Indeed, they need to ensure the number of bikes on offer is high enough to cover the residents' needs while avoiding the extra costs and space obstruction of keeping more bikes than necessary. We here deal with the first question, and aim at predicting the number of bikes rented per hour.

The Dataset¶

We will work with a dataset containing the hourly and daily count of rental bikes between the years 2011 and 2012 in Capital bikeshare system from the UC Irvine Machine Learning Repository. It was compiled by Hadi Fanaee from the Laboratory of Artificial Intelligence and Decision Support (LIAAD), University of Porto .

Feature engineering¶

Time of the day¶

Let us first divide the day into 4 equal periods: morning from 6:00 to 11:59, afternoon from 12:00 to 17:59, evening from 18:00 to 23:59, and night from 0:00 to 5:59, associated respectively with the labels 1, 2, 3, and 4. We call this column time_label.

In [8]:
def label_hour(hr):
    '''
    label the hour hr with 1 (morning), 2 (afternoon), 3 (evening), or 4 
    (night)
    
    hr: integer between 0 and 23 (included)
    '''
    if hr < 6:
        return 4
    if hr < 12:
        return 1
    if hr < 18:
        return 2
    return 3

bike_rentals['time_label'] = bike_rentals['hr'].apply(label_hour)

Conclusions¶

In this project, we trained a few simple machine learning models to predict the number of bikes rented per hour depending on weather conditions, the date, and the time of day. We found that a random forest performs significantly better than a linear regressor and slightly better than a single decision tree. This is especialy true when using all the features to train the model. We found the minimum error is obtained with 100 or more estimators. The root mean squared error is then smaller than 25% of the validation data's standard deviation, and the correlation coefficient between the prediction and data is higher than 97%.

To go further, it would be interesting to see how these models generalize to other datasets, e.g. to data from later years or different cities.
